# UKA Messenger — План реализации

Запрос огромный (фактически клон Telegram + WebRTC звонки + AI‑перевод + 3D‑логотип + 50 языков + админка). Полноценная реализация всего за одну итерацию невозможна и нереалистична — это объём команды на несколько месяцев. Предлагаю поэтапный план; начнём с прочной Фазы 1, а дальше будем итерировать по вашим приоритетам.

## Стек

- TanStack Start (уже в проекте) + React 19 + Tailwind v4
- Lovable Cloud (Postgres + Auth + Storage + Realtime) для бэкенда, авторизации, БД, файлов и WebSocket
- Lovable AI Gateway для перевода и конвертера кирилл/латин
- shadcn/ui + кастомный дизайн‑система (Glassmorphism + неон)

## Фаза 1 — Фундамент (этот заход)

1. Включить Lovable Cloud.
2. Дизайн‑система: тёмная тема, графитово‑синий фон, неоновые акценты (фиолет/ультрамарин/бирюза), glassmorphism токены и варианты кнопок/инпутов/карточек в `src/styles.css`.
3. Лендинг `/` — презентация (Безопасность, Стиль, AI‑переводчик, Конвертер), CTA «Начать», SVG‑логотип‑щит UA с металлическим градиентом и неоновым свечением (3D‑модель — отдельная фаза).
4. Авторизация:
   - Email + 6‑значный OTP через встроенную Auth (стандарт Supabase Auth — 4‑значные коды не поддерживаются провайдером; 6 цифр — отраслевой стандарт).
   - Google OAuth через брокер Lovable.
   - После первого входа — обязательная форма: Имя, Фамилия, страна, телефон, @username (с полной валидацией из ТЗ).
   - Сессия сохраняется автоматически (Supabase Auth).
5. Роли: таблица `user_roles` + функция `has_role` (security definer). Триггер при создании профиля: если телефон = `+998994902629` или email ∈ {akbarovabdulloh2012@gmail.com, alfacompofficial@gmail.com} → роль `admin`.
6. Профиль и настройки (`/settings`): редактирование имени, фамилии, @username (валидация), аватара (Storage), языка интерфейса (полный список 50 языков), смена пароля. Сохранение действительно пишет в БД (фикс бага).
7. Чат‑оболочка (`/app`): трёх‑колоночный layout — сайдбар с папками (Все/Личные/Группы/Каналы), центр (список диалогов/сообщения), правая панель (инфо).
8. Личные 1‑на‑1 чаты с real‑time через Supabase Realtime. Текстовые сообщения, поиск пользователей по @username и телефону, добавление контакта.
9. Конвертер кириллица ⇄ латиница в поле ввода (клиентская функция, без AI).
10. Админ‑панель (`/admin`, только для роли admin): список пользователей, назначение/снятие админов, просмотр групп/каналов. RLS гарантирует, что админ **не** может читать личные сообщения.
11. i18n: каркас с переключателем и полным списком 50 языков; стартовый перевод UI на RU/EN/UZ, остальные языки доступны для выбора и будут наполняться.
12. SEO: title/description/OG, sitemap.xml, robots.txt.

## Технические замечания / отклонения от ТЗ

- **4‑значный код по email** заменён на 6‑значный (Supabase OTP). 4 цифры небезопасны и провайдером не поддерживаются.
- **«Пароль на устройстве»** реализую как смену пароля аккаунта Supabase; локальный device‑pin — отдельная фишка, можно добавить позже.
- **Авто‑вход по cookies/localStorage** обеспечивается Supabase Auth сессией.
- **Полный список стран** — используем стандартный набор ISO с флагами.
- **Real‑time**: Supabase Realtime (WebSocket поверх Postgres changes) вместо отдельного Socket.IO — нативно работает с нашим стеком.
